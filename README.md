# Smart Money Dashboard

Полностью бесплатный, автономный дашборд движений хедж-фондов (13F) с наложением на твой портфель. Без серверов, без оплаты, без обслуживания.

**Как это работает:** GitHub Actions раз в день запускает Python-скрипт → скрипт тянет 13F напрямую из SEC EDGAR → сохраняет `funds.json` в репозиторий → GitHub Pages раздаёт статичный сайт, который читает этот JSON.

```
SEC EDGAR  →  GitHub Actions (cron)  →  funds.json  →  GitHub Pages
 (данные)      (раз в день, бесплатно)   (в репо)       (сайт)
```

---

## Установка за 6 шагов (~10 минут)

### 1. Создай репозиторий
Создай новый публичный репозиторий на GitHub (например `smart-money`). Публичный — чтобы GitHub Pages и Actions были бесплатны без лимитов.

### 2. Залей файлы
Скопируй всё содержимое этой папки в репозиторий и запушь:
```bash
git init
git add .
git commit -m "init smart money dashboard"
git branch -M main
git remote add origin https://github.com/ТВОЙ_ЮЗЕР/smart-money.git
git push -u origin main
```

### 3. Впиши свой email в парсер
Открой `scripts/fetch_13f.py`, найди строку:
```python
SEC_CONTACT = "your-email@example.com"
```
Замени на свой реальный email. **SEC блокирует запросы без контакта в User-Agent** — это обязательно.

### 4. Настрой фонды и портфель (опционально)
В том же файле:
- `FUNDS` — словарь `{название: CIK}`. CIK ищется на [sec.gov поиск компании](https://www.sec.gov/cgi-bin/browse-edgar?action=getcompany). Уже вписаны Berkshire, Bridgewater, Whale Rock, Octahedron.
- `MY_PORTFOLIO` — твои тикеры (для вкладки пересечений).

Если меняешь портфель — также обнови словарь `PORTFOLIO_NAMES` в `docs/index.html` (13F не содержит тикеров, только названия эмитентов, поэтому матчинг идёт по подстроке названия).

### 5. Включи GitHub Pages
Settings → Pages → Source: **Deploy from a branch** → Branch: **main**, папка **/docs** → Save.
Через минуту сайт будет доступен по адресу `https://ТВОЙ_ЮЗЕР.github.io/smart-money/`.

### 6. Запусти первое обновление вручную
Вкладка **Actions** → workflow **Update 13F Data** → **Run workflow**.
Через 1–2 минуты появится `docs/data/funds.json`, и дашборд оживёт.
Дальше он сам обновляется каждый день в 11:00 UTC.

## Уведомления в Telegram (опционально)

Дашборд может слать алерты в Telegram при каждом обновлении:
- **Дайджест 13F** — когда фонд подаёт свежий квартальный отчёт (что зашло / что слил). Шлётся только в момент новой подачи (~4 раза в год), а не каждый день.
- **Сделки инсайдеров (Form 4)** — покупки/продажи топ-менеджеров и директоров по тикерам твоего портфеля. Это самый быстрый публичный сигнал: задержка всего ~2 рабочих дня.

**Настройка за 3 шага:**
1. Напиши [@BotFather](https://t.me/BotFather) → `/newbot` → получи **токен**.
2. Узнай свой **chat_id**: напиши боту любое сообщение, затем открой `https://api.telegram.org/bot<ТОКЕН>/getUpdates` и найди `"chat":{"id": ...}`. (Или спроси у [@userinfobot](https://t.me/userinfobot).)
3. В репозитории: **Settings → Secrets and variables → Actions → New repository secret**, добавь два секрета:
   - `TELEGRAM_BOT_TOKEN` — токен бота
   - `TELEGRAM_CHAT_ID` — твой chat_id

Если секреты не заданы — всё работает как раньше, просто без уведомлений (workflow не падает).

> **Первый запуск после включения — «посев»**: скрипт запоминает текущие Form 4 и не шлёт историю. Алерты начинаются со следующей новой подачи. То же для 13F: дайджесты идут только когда `report_date` фонда реально сдвинулся.

## Инсайдеры (Form 4) — почему это ранний сигнал, а 13F — нет

13F поквартальный и с задержкой до 45 дней: если фонд начал сливать позицию в январе, ты узнаешь об этом только к 15 мая. Это карта прошлого, а не сигнал «в моменте».

**Form 4** (`scripts/fetch_form4.py`) закрывает этот разрыв: инсайдеры (CEO/CFO/директора/владельцы 10%+) обязаны сообщать о сделках со своими акциями в течение **2 рабочих дней**. Скрипт мониторит тикеры из `MY_PORTFOLIO`, ловит открытые покупки (`P`) и продажи (`S`) — «сделки убеждённости», отсеивая гранты/опционы/налоговые удержания — считает долю затронутого пакета и остаток у инсайдера, пишет их в `docs/data/insiders.json` + шлёт в Telegram. На дашборде эти сделки показываются во вкладке **🕵️ Инсайдеры (Form 4)**.

Тикеры без Form 4 (ETF вроде `KWEB`/`CLM`, иностранные ADR вроде `KC`/`CIG`/`AMKBY`) просто отсеиваются — они не подают эту форму в SEC.

**Кластеры инсайдеров** (из «особых условий»): если ≥2 разных инсайдера торгуют одну бумагу в одну сторону за 30 дней — это кластер (≥3 = сильный сигнал), он куда весомее одиночной сделки. Кластеры выводятся сверху вкладки «Инсайдеры» и шлют отдельный Telegram-алерт. **Подтверждение объёмом** (тоже «особое условие»): разжатие «пружины» на объёме ×2+ от среднего помечается как подтверждённое (`📊 объём ×N`).

Другие ранние индикаторы смены нарратива (не встроены, но куда смотреть): **13D/13G** (активисты, 5%+ пакеты), кредитные спреды **HY OAS**, срочная структура **VIX**, market breadth (% акций выше 200-дневной). Единого «фонда-индикатора» нет — 13F-гиганты подтверждают тренд с лагом, а опережают его именно инсайдеры конкретной компании.

## Вкладки дашборда
- **⚡ Пересечения с портфелем** — где фонды держат то же, что и ты, + идеи по консенсусу.
- **🧭 Темы** — портфель, сгруппированный по корзинам (крипта-майнеры, удобрения/агро, защита/сырьё, REIT, техно). Видно нетто-настроение фондов по каждой теме: куда наращивают, откуда выходят. Темы задаются в `THEMES` в `docs/index.html`.
- **🏦 Движения фондов** — топ-холдинги и покупки/продажи фондов за квартал.
- **🕵️ Инсайдеры (Form 4)** — сделки инсайдеров по тикерам портфеля (ранний сигнал).
- **📈 Макро** — ранние индикаторы смены нарратива (см. ниже).

## Макро-индикаторы (`scripts/fetch_macro.py`)

Разворот risk-on↔risk-off часто виден раньше в сырье и долларе, чем в акциях. Скрипт тянет с **Stooq** (бесплатно, без ключей) ликвидные commodity-ETF: нефть Brent/WTI (BNO/USO), газ (UNG), золото (GLD), пшеницу/кукурузу (WEAT/CORN), агробизнес (MOO), доллар (UUP). Показывается реальная цена пая, а сигнал несёт **% изменения** (день/неделя) — он повторяет базовый актив. → `docs/data/macro.json` + вкладка **📈 Макро**. При резком дневном движении (пороги в `INDICATORS`) шлёт алерт в Telegram с подсказкой, что это значит для портфеля.

> Почему ETF, а не фьючерсы: тикеры ETF на Stooq (`bno.us`, `gld.us`…) стабильны и проверяемы, а фьючерсные continuation-символы капризны. Правятся в `INDICATORS`.

> **Источник котировок (`scripts/marketdata.py`):** keyless-источники (Stooq, Yahoo) режут IP GitHub Actions — Stooq отдаёт «превышен лимит», Yahoo — `429`. Поэтому основной источник — **[Twelve Data](https://twelvedata.com/)** (бесплатный тариф: 800 запросов/день; все символы берутся **одним батч-запросом**). Stooq/Yahoo остаются keyless-фолбэком. Тем же загрузчиком пользуется и v2-анализ удобрений. Если всё молчит — на вкладке «Макро» показывается причина, а не пустой экран.
>
> **Настройка ключа Twelve Data:** зарегистрируйся на [twelvedata.com](https://twelvedata.com/) (free), скопируй API key и добавь секрет **`MARKETDATA_API_KEY`** в *Settings → Secrets and variables → Actions* (там же, где токены Telegram). Без ключа скрипты откатываются на keyless-источники (данные будут появляться с перебоями).

Честно о том, чего нет бесплатно: **бенчмарк карбамида** и **ставки танкеров** надёжного бесплатного API не имеют — они выведены в разделе «Смотреть вручную» ссылками, а не выдуманными числами.

## v2 · Удобрения: циклы + «сжатие пружины» (`scripts/fetch_fertilizers.py`)

Анализ производителей удобрений (CF, NTR, MOS, IPI, UAN, SQM, ICL) по двум осям, всё из дневных OHLC со Stooq:

**Циклы** — где бумага в своём ценовом цикле: положение в 52-недельном диапазоне, цена относительно 200-дневной средней, импульс за 120 дней. Комбинация даёт квадрант фазы: **Спад → Восстановление → Экспансия → Пик**.

**Сжатие пружины** (volatility compression / coiled spring) — ширина полос Боллинджера (BBW = 4σ/SMA20) в нижнем перцентиле своей истории = волатильность сжата, энергия копится → *пружина взведена*. Когда BBW начинает расширяться **и** цена пробивает 20-дневный диапазон — *пружина разжалась* (выстрел) вверх или вниз. На это шлётся **Telegram-алерт**; на дашборде — вкладка **🌱 Удобрения v2** с фазой цикла, статусом пружины и сводкой по сектору.

**⭐ Совпадение сигналов (confluence)** — для бумаг со «взведённой вверх» пружиной скрипт проверяет свежие инсайдерские покупки (Form 4, код `P`, переиспользуя парсер из `fetch_form4.py`). Если пружина **разжалась вверх И инсайдер покупает** — это совпадение двух независимых сигналов (техника + инсайдер), помечается ⭐ и шлёт приоритетный Telegram-алерт. Если пружина ещё только взведена вверх, но инсайдер уже покупает — мягче: ◇ «накопление, сетап зреет». Инсайдерский слой запрашивается только для up-биасных бумаг (экономия запросов) и мягко пропускается для иностранных эмитентов без Form 4.

> Это технические эвристики: они фиксируют «здесь накопилось напряжение и вот оно разрешилось», а не предсказывают направление. Не сигнал к сделке.

## Экономический цикл (`scripts/fetch_cycles.py`)

Макро-разложение делового цикла рынка по методологии вложенных циклов — **Кондратьев** (45–60 лет, качественно) / **Жюгляр** (7–11 лет, деловой цикл) / **Китчин** (2–4 года, запасы). Каждый индикатор даёт сигнал −1/0/+1; их сумма (композит) + признаки поздней фазы (инверсия кривой, рост заявок) → фаза: **Восстановление → Экспансия → Замедление → Рецессия**. К фазе привязана секторная ротация (playbook: аллокация + лидеры/отстающие). Вкладка **🌀 Циклы**; при смене фазы — Telegram-алерт.

Индикаторы: кривая доходности `T10Y2Y`, кредитный спред `BAMLH0A0HYM2`, загрузка мощностей `TCU`, безработица `UNRATE`, промпроизводство `INDPRO`, запасы/продажи `ISRATIO`, заявки `ICSA`, разрешения `PERMIT`, настроения `UMCSENT`, реальная ставка `DFII10`.

**Источник — FRED API** (`api.stlouisfed.org`). Сам graph-хост FRED (`fredgraph.csv`) и keyless-зеркала (DBnomics) с IP GitHub Actions недоступны/таймаутят, поэтому нужен **бесплатный мгновенный ключ FRED**: получи на [fredaccount.stlouisfed.org/apikeys](https://fredaccount.stlouisfed.org/apikeys) и добавь секрет **`FRED_API_KEY`** в *Settings → Secrets and variables → Actions*. Без ключа скрипт пробует keyless DBnomics/CSV (обычно не отвечают с Actions), и вкладка «Циклы» показывает причину вместо данных.

> Честно: вероятностный инструмент и иллюстрация методологии, **не инвест-рекомендация**. Фазы надёжно датируются лишь задним числом, макроданные выходят с лагом, К-цикл не имеет научного консенсуса.

## Проверка локально (без GitHub)
```bash
python3 scripts/fetch_13f.py           # создаст docs/data/funds.json
python3 scripts/fetch_form4.py         # создаст docs/data/insiders.json (инсайдеры)
python3 scripts/fetch_macro.py         # создаст docs/data/macro.json (сырьё/макро)
python3 scripts/fetch_fertilizers.py   # создаст docs/data/fertilizers.json (v2: циклы+пружина)
python3 scripts/fetch_cycles.py        # создаст docs/data/cycles.json (деловой цикл, FRED)
cd docs && python3 -m http.server 8000 # открой http://localhost:8000
```
> Примечание: SEC и Stooq могут отдавать 403 с некоторых хостингов/датацентров (в т.ч. из этого dev-окружения). С IP GitHub Actions всё работает штатно; при недоступности источника скрипты не падают и не затирают данные.

## Важные ограничения 13F (встроены в дашборд как предупреждение)
- **Нет шортов и хеджей.** 13F показывает только длинные позиции в акциях США. «Что захеджировал» из публичных данных построить невозможно.
- **Задержка до 45 дней.** Данные подаются через 45 дней после конца квартала. Это карта прошлого.
- **Нет тикеров, опционов, облигаций, иностранных акций, кэша.**

Календарь подачи: ~15 февраля (Q4), ~15 мая (Q1), ~15 августа (Q2), ~15 ноября (Q3). Реальные изменения данных происходят только в эти окна — ежедневный cron просто ловит их сразу как появятся.

## Хочешь данные качественнее / в реальном времени?
Бесплатно это невозможно — первоисточник (13F) сам по себе поквартальный с задержкой. Платные API (WhaleWisdom, Quiver Quant, sec-api.io) дают удобный JSON и иногда более частые формы (13D/G, Form 4 инсайдеров), но это уже не бесплатно и не автономно. Если решишь — скрипт легко переключить на их endpoint вместо EDGAR.

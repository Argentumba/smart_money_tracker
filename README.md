# Smart Money Dashboard

Полностью бесплатный, автономный дашборд движений хедж-фондов (13F) с наложением на твой портфель. Без серверов, без оплаты, без обслуживания.

**Как это работает:** GitHub Actions раз в день запускает Python-скрипт → скрипт тянет 13F напрямую из SEC EDGAR → сохраняет `funds.json` в репозиторий → GitHub Pages раздаёт статичный сайт, который читает этот JSON.

```
SEC EDGAR  →  GitHub Actions (cron)  →  funds.json  →  GitHub Pages
 (данные)      (раз в день, бесплатно)   (в репо)       (сайт)
```

---

## Установка за 6 шагов (~10 минут)

### 1. Создай репозиторий
Создай новый публичный репозиторий на GitHub (например `smart-money`). Публичный — чтобы GitHub Pages и Actions были бесплатны без лимитов.

### 2. Залей файлы
Скопируй всё содержимое этой папки в репозиторий и запушь:
```bash
git init
git add .
git commit -m "init smart money dashboard"
git branch -M main
git remote add origin https://github.com/ТВОЙ_ЮЗЕР/smart-money.git
git push -u origin main
```

### 3. Впиши свой email в парсер
Открой `scripts/fetch_13f.py`, найди строку:
```python
SEC_CONTACT = "your-email@example.com"
```
Замени на свой реальный email. **SEC блокирует запросы без контакта в User-Agent** — это обязательно.

### 4. Настрой фонды и портфель (опционально)
В том же файле:
- `FUNDS` — словарь `{название: CIK}`. CIK ищется на [sec.gov поиск компании](https://www.sec.gov/cgi-bin/browse-edgar?action=getcompany). Уже вписаны Berkshire, Bridgewater, Whale Rock, Octahedron.
- `MY_PORTFOLIO` — твои тикеры (для вкладки пересечений).

Если меняешь портфель — также обнови словарь `PORTFOLIO_NAMES` в `docs/index.html` (13F не содержит тикеров, только названия эмитентов, поэтому матчинг идёт по подстроке названия).

### 5. Включи GitHub Pages
Settings → Pages → Source: **Deploy from a branch** → Branch: **main**, папка **/docs** → Save.
Через минуту сайт будет доступен по адресу `https://ТВОЙ_ЮЗЕР.github.io/smart-money/`.

### 6. Запусти первое обновление вручную
Вкладка **Actions** → workflow **Update 13F Data** → **Run workflow**.
Через 1–2 минуты появится `docs/data/funds.json`, и дашборд оживёт.
Дальше он сам обновляется каждый день в 11:00 UTC.

## Проверка локально (без GitHub)
```bash
python3 scripts/fetch_13f.py          # создаст docs/data/funds.json
cd docs && python3 -m http.server 8000 # открой http://localhost:8000
```
> Примечание: SEC может отдавать 403 с некоторых хостингов/датацентров. С IP GitHub Actions всё работает штатно.

## Важные ограничения 13F (встроены в дашборд как предупреждение)
- **Нет шортов и хеджей.** 13F показывает только длинные позиции в акциях США. «Что захеджировал» из публичных данных построить невозможно.
- **Задержка до 45 дней.** Данные подаются через 45 дней после конца квартала. Это карта прошлого.
- **Нет тикеров, опционов, облигаций, иностранных акций, кэша.**

Календарь подачи: ~15 февраля (Q4), ~15 мая (Q1), ~15 августа (Q2), ~15 ноября (Q3). Реальные изменения данных происходят только в эти окна — ежедневный cron просто ловит их сразу как появятся.

## Хочешь данные качественнее / в реальном времени?
Бесплатно это невозможно — первоисточник (13F) сам по себе поквартальный с задержкой. Платные API (WhaleWisdom, Quiver Quant, sec-api.io) дают удобный JSON и иногда более частые формы (13D/G, Form 4 инсайдеров), но это уже не бесплатно и не автономно. Если решишь — скрипт легко переключить на их endpoint вместо EDGAR.
